# ブランチ名の提案

## 概要

ステージングされている変更内容を分析し、ブランチ名規約に従った適切なブランチ名を複数提案する。

---

## ブランチ名規約

- `main`: 本番環境用ブランチ（直接プッシュ禁止）
- `develop`: 開発用の統合ブランチ
- `feature/*`: 新機能開発用ブランチ
- `fix/*`: 軽微な修正用ブランチ
- `bugfix/*`: バグ修正用ブランチ
- `hotfix/*`: 本番環境の緊急修正用ブランチ
- `chore/*`: 雑務・依存関係更新用ブランチ
- `refactor/*`: リファクタリング用ブランチ
- `docs/*`: ドキュメント更新用ブランチ
- `revert/*`: コミット取り消し用ブランチ

---

## 実行手順

1. `git diff --cached` でステージングされている変更を確認
2. ブランチ名規約を読み込む
3. 変更内容に基づいて適切なブランチ名を複数提案する
